# 技术原理

这份文档解释代码里用到的每一个核心机制——为什么这么设计、算法怎么跑的、参数怎么选的。

---

## 一、世界的物理规则

### 草的生长（逻辑斯蒂方程）

每个格子有一个"当前能量"值（0 到上限）。每一轮，草按以下公式生长：

```
新能量 = 旧能量 + 生长速率 × (上限 - 旧能量)
```

这意味着：
- 空地长得快（离上限远，增量大）
- 快满的地长得慢（离上限近，增量小）
- 到上限就不长了

这就是自然界"负反馈"的最简模型。现实中草地、鱼塘、细菌培养皿都是这个规律。

### 吃草

小人站在一个格子上，按"吃的力度"从格子能量里扣。同一格子上多人同时吃，按各自力度的比例分配（抢食）。

### 移动

小人每轮可以往上下左右移动 0-2 格。每移动一格扣固定能量（移动代价）。不动不扣。

### 季节

用一个计时器循环：前 800 轮是夏天（正常生长），接下来 400 轮是冬天（生长速率为 0，草完全不长）。循环往复。

### 地形（v5）

- **肥力图**：每个格子有一个固定的"肥力"值（0 到 1），决定这个格子草最多能长到多满。肥力为 0 = 永久荒漠
- **绿洲**：圆形区域，中心肥力最高，边缘渐低
- **走廊**：连接绿洲的窄带（宽 3 格），肥力较低但不为零

地形在实验开始时生成一次，之后不变。

## 二、小人的"大脑"

### 基本架构：神经网络

每个小人的决策器是一个小型神经网络。不用任何框架，纯数学实现。

#### 输入层（感知）

小人能"看到"什么：

| 编号 | 含义 | 取值范围 |
|------|------|----------|
| 0 | 自己的余额（归一化） | 0~1 |
| 1 | 脚下的草量 | 0~1 |
| 2-3 | 视野内最肥的草在哪个方向 | -1~1 |
| 4 | 周围的平均草量 | 0~1 |
| 5 | 周围有多少人（拥挤度） | 0~1 |
| 6-7 | 最近的人在哪个方向 | -1~1 |
| 8 | 现在是不是冬天 | 0 或 1 |
| 9 | 冬天还剩多少（归一化） | 0~1 |
| 10 | 被抢过几次（归一化） | 0~1 |
| 11 | 脚下的肥力（地形属性） | 0~1 |
| 12 | 自己最近的移动量（游牧程度） | 0~1 |
| 13 | 随机噪声 | ≈0 |

#### 隐藏层（处理）

8 个"神经元"。每个神经元做一件事：把所有输入乘以各自的权重，加起来，过一个压缩函数（双曲正切，把结果压到 -1 到 1 之间）。

```
神经元输出 = tanh(输入1×权重1 + 输入2×权重2 + ... + 偏置)
```

#### 输出层（决策）

| 编号 | 含义 | 解码方式 |
|------|------|----------|
| 0 | 水平移动意愿 | > 0.3 向右，< -0.3 向左 |
| 1 | 垂直移动意愿 | > 0.3 向下，< -0.3 向上 |
| 2 | 吃草力度 | 映射到 0~1 |
| 3 | 社交力（正=给钱，负=抢钱） | -1~1 |
| 4 | 留在原地的意愿 | > 0.5 时覆盖移动输出，强制不动 |

### 前馈网络 vs 递归网络

**前馈网络（没有记忆）**

```
输入 → 隐藏层 → 输出
```

每一轮独立计算。上一轮的隐藏层状态不保留。相当于每一刻都是全新的。

**递归网络（有记忆）**

```
输入 ─┐
      ├→ 隐藏层 → 输出
上一轮的隐藏层 ─┘
```

隐藏层不仅接收当前输入，还接收自己上一轮的状态。相当于有一个"便签本"——上一步的感知会残留在里面影响这一步的决策。

数学上的区别只有一行：

```
# 前馈：
隐藏 = tanh(输入 × 权重)

# 递归：
隐藏 = tanh(输入 × 权重 + 上一轮隐藏 × 自连接权重)
```

多出来的"自连接权重"就是记忆的载体。8 个隐藏神经元互相连接 = 多 8×8=64 个参数。

## 三、演化机制

### 基因

基因 = 网络的所有权重展平成一个浮点数列表。

- 前馈网络基因长度：14×8 + 8 + 8×5 + 5 = **165 个数字**
- 递归网络基因长度：165 + 8×8 = **229 个数字**

每个数字初始从正态分布 N(0, 0.4) 随机采样。

### 自然选择

唯一的选择机制就是：**活着就能繁殖，死了就没后代。**

- 余额降到 0 → 死亡，基因从种群中消失
- 余额超过繁殖阈值 → 分裂出一个子代，子代继承父代基因的变异版本

没有"适应度函数"，没有"选择压力系数"——压力完全来自环境本身的残酷程度。

### 变异

子代基因 = 父代基因 + 高斯噪声：

```
子代基因[i] = 父代基因[i] + N(0, 0.08)
```

每个数字独立地加一个小的随机偏移。大多数变异是有害的（让策略变差），少数是有益的（让策略变好）。有益的变异通过让携带者活得更久、生得更多来扩散。

### 记忆类型的变异

每次繁殖有 1% 的概率发生"类型突变"：
- 有记忆的父代生出没记忆的子代（丢掉便签本）
- 没记忆的父代生出有记忆的子代（长出便签本）

类型突变时基因重新随机初始化（因为长度变了，旧基因不兼容）。

## 四、社交机制

### 推拉（给钱 / 抢钱）

每个小人的第 4 个输出是"社交力"，范围 -1 到 1：
- 正值 = 把自己的钱给旁边最近的人
- 负值 = 从旁边最近的人那里拉钱（抢）

**双方都有代价**：
- 发起方付代价 = |力度| × 30 × 0.3
- 抢劫时被抢方额外损失代价的一半（打架有消耗）
- 给钱时给出量 = |力度| × 30 × 0.4 × 0.5

这意味着抢劫不是无本万利——抢的过程双方都有损耗。纯抢劫策略会因为成本累积而自己垮掉。

### 为什么社交力能演化出来

虽然表面看"给钱"亏本，但如果两个携带"互给"基因的小人恰好是邻居（大概率，因为子代出生在父代旁边），互给就变成了互相保险——等价于分散风险。这不需要"识别亲属"的智能，只需要基因相近的个体在空间上聚集（亲缘选择）。

## 五、干扰实验

### 设计原理

光看种群比例（有记忆的占多少%）不够——可能只是随机漂变。需要一个直接的因果检验：**打断正常状态，看谁恢复得快。**

### 具体操作

1. 让世界跑到第 10000 轮（演化已经稳定）
2. 随机选 15 个有记忆的 + 15 个没记忆的（都是活了很久的老居民）
3. 记录它们各自正常的"每轮获能速率"（基线）
4. 烧掉它们脚下半径 4 格的草（突然失去食物来源）
5. 观察接下来 500 轮：它们的获能速率何时恢复到基线的 60%？

### 为什么这能测"意图"

- **没记忆的小人**：脚下没草了 → 看看周围 → 如果周围 2 格内有草就去吃 → 如果没有就随机游走
- **有记忆的小人**（如果记忆被"用对"了）：脚下没草了 → 脑子里还存着"东边有绿洲" → 朝东走

区别在于：没记忆的只能做局部搜索（看得到才去），有记忆的能做定向迁徙（看不到也知道往哪走）。后者恢复应该更快。

### 为什么选"烧地"而不是"传送"

早期设计考虑过把小人传送到地图另一端。问题：
- 传送到的地方可能恰好是满格草地 → 测不出恢复能力
- 有记忆的小人脑子里存的是老位置的信息 → 传送后旧记忆全是错的，反而成为干扰

烧地更干净：小人还在原地，环境突然恶化，考验的是"会不会主动离开、往哪走"。

## 六、统计方法

### 多种子重复

每组实验跑 5-20 个不同的随机种子。同一个设计在不同种子下结果可能差很大（随机漂变），所以需要多次重复来区分"系统性趋势"和"运气"。

### 韦尔奇 t 检验

比较两组数据的均值是否有显著差异。公式：

```
t = (均值A - 均值B) / √(方差A/样本数A + 方差B/样本数B)
```

t 值越大，差异越不可能是偶然。一般 |t| > 2 认为显著，> 3 认为极显著。

不用配对 t 检验是因为两组样本量可能不等（有些种子里某一方灭绝了）。

### 显著性标记

| 标记 | 含义 |
|------|------|
| ns | 不显著（|t| < 1.7） |
| * | 弱显著（|t| 1.7~2.5） |
| ** | 显著（|t| 2.5~3.5） |
| *** | 极显著（|t| > 3.5） |

## 七、参数选择的逻辑

| 参数 | 值 | 为什么选这个 |
|------|-----|-------------|
| 种群上限 | 200~400 | 太小演化靠运气不靠选择，太大跑得慢 |
| 初始余额 | 300~500 | 够活几百轮，给演化启动时间 |
| 繁殖阈值 | 800~1200 | 要比初始余额高很多，确保只有"过得好"的才繁殖 |
| 视野 | 2~3 格 | 太大则信息充足不需要记忆；太小则随机游走效率太低全灭 |
| 移动代价 | 2~3/格 | 让"到处跑"有成本，但不至于"走一步就快饿死" |
| 变异幅度 | 0.08~0.15 | 太小演化太慢，太大好基因被噪声淹没 |
| 冬天长度 | 0~400 轮 | 0 = 无压力；400 = 极限压力（现有结果表明 400 太长了） |

## 八、代码结构

```
simulation_v5.py
├── 超参数定义
├── 地图生成（generate_terrain）
│   └── 放绿洲 → 画走廊 → 返回肥力图
├── 神经网络（forward_ff / forward_rnn）
│   └── 纯手写矩阵乘法 + tanh 激活
├── Agent 数据结构
│   └── 位置、余额、基因、隐藏状态、统计计数器
├── World 类
│   ├── step()：一轮的全部逻辑
│   │   ├── 草生长
│   │   ├── 基础代谢扣费
│   │   ├── 感知 → 神经网络 → 决策
│   │   ├── 移动 + 扣费
│   │   ├── 吃草 + 分配
│   │   ├── 社交推拉
│   │   ├── 死亡判定
│   │   └── 繁殖 + 变异
│   ├── run_disturbance()：干扰实验
│   └── collect_results()：汇总数据
└── main()：跑多个种子 → 统计 → 输出
```

无外部依赖。所有数学（矩阵乘法、tanh、统计检验）都用 Python 标准库的基本运算手写。
